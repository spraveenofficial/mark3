Getting Started with GIT in 2021

Praveen Singh

Date: 12th August 2021

Hey Folks, have you wondered what Git is, what is works for and how to get started with?

Well, as we all developers already knows that GIT is very popular tool for developers around the globe. If you don't know what what GIT exactly is, then don't worry. This article will try to clear all such doubts that comes in every begineer mind. Let't jump to main topic.

Prerequisites

  • Internet & Computer Basics
  • Terminal Basics - clear, cd , mkdir, rm

    • That’s pretty much. You are ready to go :)

    What is Git?

    Let's know this from Git's documentation:

    Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

    The full form of GIT is Global Information Tracker. It is basically designed for programmers to work together It is basically designed for programmers to work together on the same source code at the same time.

    So, imagine you are working in an company / organisation with 100+ people and all the developers have a copy of the project on their local system of the computer. Now, if a second developer who is also your friend, let's call him Rohit, introdues a new changes in database schemas and tells you about it because he is your friend only, the project with new commit will only be stored in Rohit's system (local machine) and you as the lead developer will have to copy Rohit's local machine through some resources. Now this situation becomes worse. Here git plays very important role. When each and every employee is use GIT, and if Rohit makes any changes to codebase, you along with other developers will find that out instantly. What change that some made, each and every thing gets noticed by using any version control system.

    Getting Started with Git

    Before understanding and using Git commands, I am assuming that git is already installed and configured on your machine correctly.

    If not, Please follow the steps below to install and configure Git.

    Step 1: Installing Git

    For installing Git locally to your machine, Please follow this Link to install Git on your System.

    Once you’ve installed Git on your system, let’s fire up the Terminal/Command Prompt and get ready to go with Git. Please run the following command to check if Git was installed properly or not. If this commands not works for you please try to add Environment Variable to your system.

    git --version

    Step 2: Configuring Git

    It's very important to set up some of the basics credentials like email and name before using git, by this you will gets to know which changes was made by whom.

    First, run the below given command to setup your username.

    git config --global user.name "yourusername"

    And now, run the following command to set up your email:

    git config --global user.email "email@example.com"

    Basic Git Commands

    Till Now, that we have basic knowledge of what Git does and what's the main purpose of Git. We have it installed on our system, let’s start using it.

    Git Init

    This command is used to initialized an empty git repository to your current directory.

    $ git init

    Git Status

    It gives the status of current working directory files changed and those which still need to add or commit. It does this on the basis of changes made in the files.

    $ git status

    Git Add

    This is use to add one or more files to the staging area.

    $ git add < filename >

    This is use to add all of the files to the staging area.

    $ git add .

    Git Commit

    A commit is nothing but its like a checkpoint for any repository. This command is used to commit changes to the head locally.

    $ git commit -m "commit message"

    -m flag is used to give a message related to the commit.

    Git Branch

    While you are working on any project, there might be a point say where you want to try adding new features to your already awesome project.

    $ git branch

    Git Log

    This command is used to get all the previous commit with each and every details like message, date & time.

    $ git log

    Git Push

    This commands Push the changes to the remote repository.

    $ git push origin master
